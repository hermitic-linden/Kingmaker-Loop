# ARCHITECT.md

> **SOP 循环的运行时压缩内存**
>
> 本质：不是日志，不是 checklist，是可执行快照（executable snapshot）。
> 用途：截断恢复、上下文锚定、执行推进的唯一依据。
> 读取顺序：自上而下，STATUS ≠ NOT REACHED 的最顶层即为当前执行点。
>
> ***
>
> ## 写入契约
>
> 每一层必须遵守以下三条规则：
>
> 1. **状态与内容绑定** — 禁止只写 `STATUS: DONE` 而不写"做了什么"
> 2. **每层有最小产出物** — 空占位符不超过 3 行（见下方 Schema）
> 3. **层间数据流可追踪** — 上游输出必须是下游输入，禁止凭空创造
>
> ***

## STATE

### 运行时指针

| 字段               | 值                     | 说明            |
| ---------------- | --------------------- | ------------- |
| `CURRENT_LAYER`  | `INIT`                | 当前所在 SOP 层    |
| `CURRENT_STEP`   | `—`                   | 当前层内的子步骤      |
| `CYCLE_COUNT`    | `0`                   | 完成的完整 SOP 循环数 |
| `LAST_ACTION`    | `文件初始化`               | 最后一次写入动作      |
| `LAST_TIMESTAMP` | `YYYY-MM-DDT__:__:__` | 上次写入时间戳       |
| `RESUME_POINT`   | `STATE`               | 若上下文丢失，从此行恢复  |

### 状态机定义

```
INIT → OBSERVE → MODEL → PLAN → EXECUTE → REVIEW → (回到 OBSERVE 为新一轮)
  ↓        ↓         ↓        ↓          ↓         ↓
 BLOCK   BLOCK     BLOCK    BLOCK      BLOCK     BLOCK
```

**转移守卫（Guard Conditions）：**

| 转移               | 前置条件                              | 后置动作                              |
| ---------------- | --------------------------------- | --------------------------------- |
| INIT → OBSERVE   | `CURRENT_LAYER == INIT` 且收到具体任务指令 | 清空所有 `_ [等待填充]` 占位符               |
| OBSERVE → MODEL  | OBSERVE 层 ≥ 3 个非空产出项              | 将 OBSERVE 的"差距"转为 MODEL 的"变量筛选范围" |
| MODEL → PLAN     | MODEL 层 DAG 至少包含 2 条已验证边          | 将 MODEL 的"可操作变量"注入 PLAN 的"通路候选"   |
| PLAN → EXECUTE   | PLAN 层冲突目标已解决（低危继续/高危交人）          | 按里程碑拆分子任务，写入 EXECUTE 任务树          |
| EXECUTE → REVIEW | EXECUTE 层所有里程碑状态 ≠ NOT\_STARTED   | 收集产物，准备审计                         |
| REVIEW → OBSERVE | REVIEW 层偏差分析完成                    | 新一轮循环，差距缩小则减量，扩大则增粒               |
| 任何层 → BLOCK      | 连续 3 次重试失败                        | 标注 BLOCK，交人类决策                    |

### 检查点摘要

| 检查点          | 通过条件            | 当前状态               |
| ------------ | --------------- | ------------------ |
| CP-1: 任务指令明确 | 用户输入包含可操作的领域/目标 | `_pending_`        |
| CP-2: 备份就绪   | 执行前 `.bak` 已创建  | `_not_needed_yet_` |
| CP-3: 工具可用   | MCP/Skill 列表已加载 | `_loaded_at_init_` |
